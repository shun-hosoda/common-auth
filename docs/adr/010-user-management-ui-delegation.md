# ADR-010: ユーザー管理UI実装方針

## ステータス

Accepted (2026-03-01)

## コンテキスト

Phase 3でtenant_adminによるユーザー管理機能を実装するにあたり、UIの実装方針を決定する必要があった。

### 選択肢

**Option A: カスタムReact実装**
- Backend Admin APIラッパーを実装し、React UIでユーザー一覧・CRUD操作を提供
- 利点: フルコントロール、UIの一貫性
- 欠点: 実装工数が大きい（Backend API + React UIで推定2〜3週間）

**Option B: Keycloak管理コンソール委譲**
- ユーザー管理操作をKeycloak管理コンソールのURLへリダイレクト
- 利点: 実装工数少（数時間）、Keycloakの豊富な機能をそのまま活用
- 欠点: UIカスタマイズに制限あり

## 決定

**Option B（Keycloak管理コンソール委譲）を採用する。**

MVPフェーズでは工数削減を優先し、Keycloak管理コンソールへのディープリンクでユーザー管理を提供する。フロントエンドからは`/admin/users`画面に管理コンソールへのリンクボタンを配置する。

将来的なカスタムUI実装はPhase 4以降で検討する。

## 影響

- フロントエンドの実装工数が大幅に削減される
- UIデザインの一貫性はフルカスタムより劣るが、機能面では同等以上
- Keycloakのバージョンアップに伴う管理コンソールURLの変更を環境変数で吸収する（`VITE_KEYCLOAK_URL`）
