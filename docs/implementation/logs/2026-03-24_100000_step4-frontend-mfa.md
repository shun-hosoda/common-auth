## 合意事項

1. フロントエンドテストなし（React Testing Library 未導入 → 手動検証）
2. Email OTP ラジオは **disabled + 「準備中（KC 26以降で対応予定）」** で表示
3. MFA **有効化時（false→true）** および **無効化時（true→false）** に **確認ダイアログ** を表示
   - 有効化: 「MFAを有効にすると、テナント内の全ユーザーに次回ログイン時からMFAが要求されます。」
   - 無効化: 「MFAを無効にすると、テナント内の全ユーザーのMFA要求が解除されます。」
4. dropdown「セキュリティ設定」は管理者のみ `/security` へ遷移、一般ユーザーは `configureMFA()` 維持
5. MfaStatusCard は全ユーザーの Dashboard に表示（`/auth/mfa-status` は認証済み全ユーザーアクセス可）
6. `adminApi.ts` の `request` ヘルパーに `baseOverride` パラメータを追加し、auth router (`/api/auth`) 呼び出しでもエラーハンドリングを統一する
7. MfaStatusCard は内部で `useAuth().getAccessToken()` を使用してトークンを取得する（既存の AdminUsers パターンと統一）

## API仕様（Frontend視点）

### GET /api/admin/security/mfa → MfaSettings
```json
{ "mfa_enabled": false, "mfa_method": "totp" }
```

### PUT /api/admin/security/mfa → MfaUpdateResult
Request: `{ "mfa_enabled": true, "mfa_method": "totp" }`
Response: `{ "status": "updated", "mfa_enabled": true, "mfa_method": "totp", "users_updated": 5, "users_failed": 0 }`

### GET /api/auth/mfa-status → MfaStatus
```json
{ "mfa_enabled": true, "mfa_method": "totp", "mfa_configured": false }
```

## 検証手順

1. `npm run dev`（react-app）+ `uvicorn`（fastapi-app）
2. `admin_acme-corp@example.com` でログイン
3. SideNav に「セキュリティ設定」が表示される
4. `/security` に遷移 → 初期値 mfa_enabled=false, mfa_method=totp
5. トグルON → 有効化確認ダイアログ → 保存 → 成功メッセージ
5b. トグルOFF → 無効化確認ダイアログ → 保存 → 成功メッセージ
6. Dashboard → MfaStatusCard が「MFA 要設定」表示
7. 一般ユーザーで SideNav に「セキュリティ設定」が非表示
8. 一般ユーザーの Dashboard で MfaStatusCard が表示される
